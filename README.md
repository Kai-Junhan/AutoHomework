
# AutoHomework

#### 批量处理 Word 格式前端作业、JS 实验报告，自动填充个人信息 + Kimi 生成合规答案 / 报告内容，模拟人工手写格式。

（其实就是作者被学习报告作业整无奈了，仅处于能用的水准）

### 一、环境准备

#### 安装 Python 3.x 环境（推荐 3.8+）

打开终端 / CMD，执行以下依赖安装命令：

```bash
pip install python-docx requests
```

### 二、必做配置修改（仅需改一处！）

1. 打开 `config.py` 文件
2. 填写以下内容：
   - 替换为你的 **Kimi API Token**（从月之暗面开放平台获取）
   - 填写真实个人信息（姓名、学号、专业班级）
   - （可选）自定义输入输出文件夹名称
3. 保存 `config.py`，配置即可生效

### 三、使用步骤

1. 在脚本同级目录下新建文件夹，命名为 `js_homework_input`（或你在 `config.py` 中自定义的输入文件夹名称）
2. 将需要处理的空白 Word 文档（仅支持 `.docx` 格式）放入该文件夹
3. 运行对应脚本：

```bash
# 处理普通作业（选择题/简答题）
python homework_auto.py

# 处理实验报告（自动生成 Result + Conclusion）
python experiment_auto.py
```

生成的完整文件会自动保存到 `js_homework_answered` 文件夹（或你在 `config.py` 中自定义的输出文件夹名称），**不会覆盖原文件**。

### 四、核心特性

- 配置统一管理：仅需修改 `config.py` 一处，两个脚本同步生效
- 自动识别题目 / 报告填充位置，精准匹配格式
- 答案 / 报告内容专业合规，符合大学作业要求
- 实验报告 Conclusion 严格控制在 80 词以内，自动截断超长内容
- 图片题、无效内容自动留空，无明显机器生成痕迹
- 支持批量处理多个文档，运行过程不中断
- 最大程度保留原文档的所有格式（字体、行距、表头等）

### 五、注意事项

- 请勿公开泄露 `config.py` 文件，其中包含你的隐私信息（API Token 和个人信息）
- 脚本仅用于学习辅助，请勿用于违规用途
- 确保网络通畅，API 请求失败时会在终端提示并跳过对应内容
- 仅支持 `.docx` 格式文档，其他格式（如 `.doc`、`.pdf`）不支持
- 功能处于「能用」水准，若遇到特殊格式文档或生成异常，可手动微调最终结果
